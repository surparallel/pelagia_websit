Pelagia reference manual Lua language edition

Introduce

Pelagia is an automatic parallelization tool developed by surparallel open source based on the concept of sequential virtual machine.

Pelagia is developed by ANSI C, abides by agpl protocol, and supports multiple operating systems and languages.

It supports automatic parallelization, transparent thread calling, embedded key value database, embedded multi language running environment, and provides APIs and documents for each language.

The set concept in Boolean algebra is used to describe the operation of data. Including key, value, ordered set, index set and other types.

 

Who is suitable to read this manual?

This manual is applicable to professional software developers, through which you can accurately understand the details of Pelagia development.

 

What do you need to know before reading this manual?

Before reading this manual, you need to know the basic knowledge of LUA language, Boolean algebra, data structure, etc.

 

Related resources

Pelagia website: https://surparallel.org

 

Environmental

There are two ways to use Lua on Pelagia, one is through C embedding, the other is to use Pelagia executable program to run. Pelagia executable program has rich startup parameters and command line. They can be found by launching the command C h and the interactive command help. Support lua5.1 and luajit2.0.

Environmental installation

Install on Linux system

Linux &amp; Mac installation is very simple, just download the source package and decompress and compile it on the terminal. This article uses version 0.1 for installation:

git clone https://github.com/surparallel/pelagia.git
cd pelagia/src
make linux

 

Install on Mac OS X

git clone https://github.com/surparallel/pelagia.git
cd pelagia/src
make macosx

 

Install on window system

After decompression, enter the msvcs directory and open pelagia.sln for compilation.

dbPath: Database save directory.

core: Number of threads.

logOutput: Type of log output. " file" of "print"

logLevel: Level of log output. number of 0 ~ 6.

enum OutPutLevel

{

    log_null = 0,

    log_error,

    log_warn,

    log_stat,

    log_fun,

    log_details,

    log_all,

};

 

maxTableWeight: every file of table max weight. Default 1000.

luaPath: The directory of lua script defaults to the current directory.

luaDllPath: The directory of lua dynamic library, which is the current directory by default.

dllPath: Dynamic library load directory, default to current directory.

luaHot: Whether it is a hot update. The default value is 0, which means it is closed. Every time you restart the loading script.

nosave: Do not save data to file.

stat: Enable statistics,Use when loglevel is 3.

checkTime: The interval time of statistics is 5 seconds by default.

maxQueue: The maximum length of the message queue. If it exceeds the length, it will be discarded. The default value is 0, which means it is closed.

 

Start ./pelagia and enter the command

>iwj config.json

 

You can call base.lua for basic testing

>rcj base

Event API

Function module for asynchronous communication between user and system. Created in user state. Used to send messages inside the system to outside the system. It is not recommended that users use this module to send messages inside the system.

 

EventSend

Send data to message queue.

Grammar

string EventSend(string handle, string v);

Parameter

handle: Event handle encrypted by Base64.

v: Message value

Return

NULL
